# Code Generation

Code generation is orchestrated by `CharismaGenerator` and writer components under `src/Charisma.Generator/Writers`.

## Generator Inputs

- validated `CharismaSchema`
- `GeneratorOptions`:
  - `RootNamespace`
  - `GeneratorVersion`
  - `OutputDirectory`

## Output Structure

Generated artifacts include:

- `Enums/*.g.cs`
- `Models/*.g.cs`
- `Delegates/*Delegate.g.cs`
- `Metadata/ModelMetadataRegistry.g.cs`
- `Args/*Args.g.cs`
- `Filters/*Filter.g.cs`
- `Select/*Select.g.cs`
- `Omit/*Omit.g.cs`
- `Include/*Include.g.cs`
- `CharismaClient.g.cs`

## Writer Responsibilities

- `EnumWriter`: enum type emission
- `ModelWriter`: model POCOs and scalar mapping
- `DelegateWriter`: delegate methods and convenience APIs
- `MetadataWriter`: runtime metadata registry
- `ArgsWriter`: operation args and payload classes
- `FilterWriter`: where and scalar filter types
- `SelectWriter`: projection masks
- `OmitWriter`: omission masks and global omit options
- `IncludeWriter`: include graph types
- `ClientWriter`: root `CharismaClient`

## Generated Headers and Determinism

Every generated file contains:

- auto-generated warning header
- generator version
- schema hash
- `#nullable enable`

This supports:

- reproducibility
- regeneration consistency
- debug traceability

Typical delegate methods:

- `FindUniqueAsync`

- `UpdateAsync`

- `UpdateManyAndReturnAsync`
- `DeleteAsync`
- `DeleteManyAsync`
- `UpsertAsync`
- `CountAsync`
- `AggregateAsync`

Convenience methods are generated based on key shape (single/composite primary key and selector support).

## Type Mapping Notes

Core scalar-to-C# mapping includes:

- `Id`/`UUID` -> `Guid`
- `String` -> `string`
- `Int` -> `int`
- `Float` -> `double`
- `Decimal` -> `decimal`
- `Boolean` -> `bool`
- `DateTime` -> `DateTime`
- `Json` -> `Json`
- `Bytes` -> `byte[]`

Optional/list flags map to nullable/list forms.

## Regeneration Workflow

Recommended workflow and examples

<!-- markdownlint-disable MD032 MD031 MD029 -->

1. Edit `schema.charisma` to add or change models.
2. Run the generator:

```bash
charisma generate --schema schema.charisma --output ./src/MyApp.Generated
```

3. Build your solution to compile the generated assembly.
4. Update application code to use new types and iterate.
<!-- markdownlint-enable MD032 MD031 MD029 -->

Example generated files

- `Generated/Models/Post.g.cs` — model POCOs
- `Generated/Delegates/PostDelegate.g.cs` — model delegate with `FindManyAsync`, `CreateAsync`, etc.
- `Generated/Args/PostFindManyArgs.g.cs` — args and filter types
- `Generated/CharismaClient.g.cs` — client root and wiring

Sample generated signature (illustrative):

```csharp
public partial class PostDelegate
{
  public Task<Post?> FindUniqueAsync(PostFindUniqueArgs args);
  public Task<IEnumerable<Post>> FindManyAsync(PostFindManyArgs args);
  public Task<Post> CreateAsync(PostCreateArgs args);
}
```

Determinism and headers

Each file includes an autogenerated header with generator version and a schema hash so builds are reproducible and regeneration diffs are traceable.

Generator flags and options

- `--output` — where to write generated code.
- `--root-namespace` — override the emitted namespace when needed.
- `--force` — overwrite existing files (use with care).

Testing and CI

- Include `charisma generate` in your build pipeline where generated code must be produced before compilation.
- Keep the schema in source control; use the schema hash in CI artifacts to verify generator inputs.
